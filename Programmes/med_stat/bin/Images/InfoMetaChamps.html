Principe du module statistique de MedinTux

Avant toute étude statistique il convient :
De sélectionner les données à analyser, et de définir les critères d'inclusion et de sélection statistiques des dossiers. C'est le rôle des filtres :
Le filtre de sélection SQL (donnés SQL).
Le Méta-filtre (données non SQL).
 Puis de dénombrer les données à analyser dans les dossiers sélectionnés par les filtres, c'est l'objet des classes de dénombrement qui seront chargées de compter les données selon les critères de dénombrement que vous aurez choisis.


Le Filtre de sélection SQL
Le filtre de sélection SQL permet d'opérer plusieurs tests sur les données des champs SQL de MedinTux afin de déterminer et filtrer les dossiers à inclure dans l'analyse statistique. Dans ce filtre Il vous sera donc demander de sélectionner les champs SQL à évaluer ainsi que les test à opéréer dessus. A droite du filtre, une zone de texte affiche la requète SQL produite par le filtre.

Méta-champs de données
Un méta-champ est une données extraite des documents textes. ils peuvent tout comme les champs SQL faire l'objet d'opérations de comparaison et autres opérations logiques aussi bien pour sélectionner les dossiers à inclure dans une analyse que pour le dénombrement de critères statistiques.
La liste de définition des méta-champs : permet de définir leur nom ou libellé, le document source texte à analyser "Observation" 'Prescription", ansi que la méthode d'extraction des données dans ce texte.

Le Méta-Filtre
Le méta filtre est une série de tests sur les méta-champs permettant de valider et sélectionner les dossiers à inclure selon le résultat des tests
Il a la même structure que celui du filtre de sélection SQL situé au dessus de lui. La seule différence est que les tests portent sur des données provenant des méta-champs et non de champs SQL. Le nom du méta-champ à inclure dans l'analyse est donc à renseigner à partir de ceux disponibles de la liste à droite, dans la boîte de définition des méta-champs.


Classes de dénombrement
Les classes de dénombrement définissent la façon dont doivent être dénombrées les données à analyser. Elles comportent un Nom de classe qu'il vous est libre de définir, c'est lui qui sera dénombré et apparaîtra dans le résultat final.
Il faut aussi lui donner le Champ évalué (méta-champ ou champ SQL) qui sera dénombré.


La liste des résultats
Cette liste regroupe les lignes de données qui répondent positivement aux tests de sélection. Cette liste peut être exportée selon différents formats afin de l'exploiter dans d'autres logiciels Libre office & consorts.
