............................... POLITIQUE DE GESTION DES FICHIERS .INI ............................................

° -a- Le fichier de configuration est d'abord recherché : au niveau de l'exécutable, nom_de_lexecutable.ini puis on y recherche :



- Une mention d'indirection de fichier du type :

[PathIni]

RedirectPath =../../IndirectConfig.ini où 'IndirectConfig.ini' est le fichier dont il sera tenu compte.



- Une mention d'indirection selon adresse IP ou adresse MAC permettant à un poste de travail distant exécutant

le binaire chargé sur un serveur applicatif de rechercher ses fichiers de configuration dans un répertoire lié

à son adresse IP ou adresse MAC sur ce serveur d'applicatifs.

- Cette disposition est utile lorsque des postes de travail disk less PXE lancent l'applicatif

à partir d'un serveur applicatif distant type serveur PXE ou autre.

- Cela permet à chaque poste de travail distant disk less qui ne disposent pas d'espace

de stokage permanent, de disposer de leur propre fichier de configuration sur le serveur

applicatif distant.

- Ces espaces sont définis au niveau du serveur applicatif par une syntaxe obéissant aux règles suivantes :



;......... cas d'une indirection selon adresse IP ....................



[PathIni]

PathOnIP = ../../IndirectConfig/ , _ , poste_



;......... cas d'une indirection selon adresse MAC ....................

; ici nous voyons que le séparateur n'est pas indiqué auquel cas celui par défaut, le point sera utilisé

; de même le préfixe n'étant pas indiqué il ne sera pa utilisé.



[PathIni]

PathOnMacAdr = ../../IndirectConfig/



* le premier est le répertoire (peut être indiqué en mode relatif) dans lequel retrouver le répertoire des

fichiers de configuration d'un poste de travail liés à son adresse IP ou son adresse MAC.

* le deucième paramètre est un séparateur, qui remplacera les points composant l'adresse IP du poste de travail

ou les deux points dans l'adresse MAC (selon PathOnIP ou PathOnMacAdr)

exemple : 192.168.100.79 l'adresse IP deviendra 192_168_100_79 si le séparateur est _

exemple : 00:0C:29:CD:78:F0 l'adresse MAC deviendra 00.0C.29.CD.78.F0 si le séparateur est . ou non indiqué (point par défaut)



* le troisième est le préfixe du nom du répertoire des fichiers de configuration liés au poste de travail .



Au total le nom du répertoire de configuration lié à un poste distant sera :

Préfixe + adresse IP ou MAC du poste ( dont les points sont remplacés par le deuxième paramètre, ou si non indiqué un point)

spécifiant le répertoire dans lequel l'on recherchera les fichiers de configuration qui lui sont liés



../../IndirectConfig/ répertoire global des fichiers de configuration distants

(le point de départ étant sur le poste du serveur applicatif, le répertoire du binaire exécuté)

poste_192_168_100_79 répertoire des fichiers de configuration d'un poste distant situé à l'adresse IP : 192.168.100.79

Manager.ini les différents fichiers de configuration de ce poste distant

drtux.ini

biogest.ini



° -b- Si echec le fichier de configuration sera recherché en : ../../../config/nom_de_lexecutable.ini

° -c- Si echec le fichier de configuration sera recherché en : homePath/.organizationName/organizationDomain/nom_de_lexecutable.ini

° -d- Si echec le fichier de configuration sera recherché en : ../../../config/MedinTuxGlobal.ini

° -e- Si echec le fichier de configuration sera recherché en : homePath/.organizationName/organizationDomain/MedinTuxGlobal.ini

° -f- Si echec le fichier de configuration sera recherché en : ../../../MedinTuxGlobal.ini
